## Week 1 · 立项（v0.1 已完成 ✓）

**目标**：把现有原型变成公开的开源项目。

**交付物**：
- [x] HTML 原型（灵感卡片捕获、随机回顾、统计）
- [ ] 建立 GitHub 仓库（公开）
- [ ] 上传 README、MANIFESTO、ROADMAP、ARCHITECTURE
- [ ] 部署到 GitHub Pages 或 Vercel，拿到公开访问链接
- [ ] 写第一篇博客《为什么我要养育一个陪我创业的工具》并发出去

**博客#1 核心信息**：
- 我的处境（孵化器、智能运动硬件、迷茫）
- 现有工具为什么不够
- 我想做一个什么样的东西
- 邀请同类人围观或参与

**衡量成功**：
- 仓库可以被搜索到
- 至少 3 个真实人看了博客并给出反馈
- 自己每天用一次 v0.1

## Week 2-3 · 洞察舱（v0.2）

**目标**：让"信息进来 → 变成可检索的洞察"这条流水线跑通。

**核心新功能**：

### 📥 多源输入
- 文字输入（已有）
- 截图粘贴 + OCR（Tesseract.js 浏览器内执行）
- 链接抓取（先做最简单的：粘贴 URL → 提取标题和正文 → 让你确认）
- 语音录入（浏览器 Web Speech API，免费、本地）

### 🧹 拾光人代理
- 接入 Claude API
- 收到长内容时，自动提炼成 1-3 个原子想法
- 自动推荐标签
- 自动找出与已有想法的关联

### 🕸️ 神经元图谱
- D3.js 力导向图
- 节点 = 原子想法，连线 = 共享标签或语义相似
- 点击节点查看详情
- 按主题聚类

### 🎲 跨界推送（这是关键差异化）
- 你设置自己的"主领域"
- 每周三次，拾光人从一个**正交领域**抓一段内容推给你
- 推送理由会告诉你"为什么觉得这跟你有关"

**衡量成功**：
- 我自己累积 50+ 个原子想法
- 神经元图谱能看出 3 个以上明显聚类
- 至少有 1 次"跨界推送"让我感叹"还真有关联"

**博客#2 主题**：《我把 100 条素材扔给拾光人，它给了我一张自己的地图》

## Week 4 · 视频日记（v0.3）

**目标**：把"对着镜头说话"变成可检索的洞察。这是 Lantern 最独特的能力。

**核心新功能**：

### 🎬 视频上传与处理
- 拖拽视频文件进来
- 调用 Whisper API（OpenAI 开源转录模型）转录
- 保留时间戳

### 🧠 视频洞察提取
- 拾光人读转录文本
- 自动提取核心观点、矛盾点、情绪
- 每个洞察都附带视频时间戳，点击跳回原片段

### 🪞 "和过去的自己对话"
- 在洞察卡片旁有按钮："这条想法跟我之前说过的有冲突吗？"
- 点灯人会调取相关历史想法做对比
- 这是 Lantern 第一次展现"个人镜子"的能力

**衡量成功**：
- 我上传过去录的 5+ 个反思视频
- 每个视频被提炼出 3-10 个洞察卡片
- 至少发现 1 次"我前后矛盾的地方"或"反复出现的主题"

**博客#3 主题**：《我把过去三个月的视频日记喂给 AI，它指出了三件我没意识到的事》

## Week 5 · 灵感舱 + 点灯人（v0.4）

**目标**：从"被动接收信息"升级到"主动用 AI 帮自己想问题"。

**核心新功能**：

### 💭 问题入口
- 三种问题：
  1. **闭合型**（明确的困惑，需要思考方向）
  2. **开放型**（好奇心，没具体问题）
  3. **观察型**（看到周围人发生的事，引发的感想）

### 🔥 点灯人代理
- 接到问题后，先做三件事：
  1. 从知识库调取最相关的 5-10 条历史想法
  2. 反问澄清（如果问题不够清晰）
  3. 给出"过去的你 + 通用知识"融合后的视角
- 关键：**它先反问，不急着给答案**

### 🎯 问题与产出的关联
- 闭合型问题最终会归档到"输出舱"
- 开放型问题如果只是观察，会自动并入知识库
- 观察型问题被标记为"等待发酵"，定期由拾光人提醒回看

**衡量成功**：
- 我每周主动用点灯人 5+ 次
- 至少 3 次"被反问得很爽"的体验（不是我想要的答案，但是好问题）
- 至少 1 个困惑因此被想清楚

**博客#4 主题**：《我让 AI 看着我过去半年的笔记跟我吵架，结果出乎意料》

## Week 6 · 输出舱 + 守灯人（v0.5）

**目标**：闭环。让所有沉淀转化成持续的对外输出。

**核心新功能**：

### 📤 输出追踪
- 三类输出：
  1. **博客 / 文字**：从点灯人对话或洞察卡片一键生成草稿
  2. **视频 / 音频**：录制并自动转录，转录文本可二次编辑成文案
  3. **硬件 / 产品**：里程碑记录（不是详细任务管理，是节奏跟踪）

### ⛵ 守灯人代理
- **节奏督促**：草稿超过 14 天没发，主动提醒
- **70 分判断**：你说"这还不够好"，它说"给我看看，让我判断"
- **情绪陪伴**：识别你的疲惫和兴奋，给恰当的回应（不是无脑鼓励）
- **build in public 仪式感**：发布时它会写一条简短的"庆祝语"

### 🔁 闭环动力
- 输出发布后，反馈（评论、转发数等）作为新的"洞察"流回洞察舱
- 形成完整闭环：洞察 → 灵感 → 输出 → 反馈 → 新洞察

**衡量成功**：
- 我已经发布了至少 5 篇 build-in-public 博客
- 至少 1 次"想拖延但被守灯人推着发出去"的经历
- 智能帆板创业项目有 1-2 个具体里程碑被跟踪

**博客#5 主题**：《灯亮起来了——Lantern v0.5 与我这 6 周的变化》

## 6 周之后

如果一切顺利，到第 6 周末你会拥有：
- 一个完整可用的个人成长工具
- 5 篇 build-in-public 博客 + 一个有内容的 GitHub 仓库
- 一群知道你在做什么的早期围观者
- 最重要的：**一个比 6 周前更清楚自己想要什么的自己**

接下来可能的方向：
- **深耕**：基于这 6 周的真实使用反馈，重点优化最有用的那一两个模块
- **拓展**：加入更多输入源（播客转录、Twitter 收藏、Kindle 笔记导入）
- **社区化**：如果有用户主动来用，做一个"种灯人社区"（保留单机数据隔离，但分享 公共方法论）
- **数字克隆**：把所有数据训练一个 RAG 模型，做真正的"高阶版本的你"

但所有这些都不是现在该想的事。**现在该想的事只有：能不能下周发出 v0.2**。

## 节奏提醒

每完成一个 milestone：
1. 守灯人发出"庆祝信号"
2. 写博客
3. 把博客链接发到一个朋友 / 一个社群 / 一条推文
4. 收集反馈，整理成新的"洞察"喂回洞察舱
5. 进入下一个 milestone

如果某个 milestone 拖延超过 1 周，停下来诚实问自己：
- 是这个功能本身不对吗？→ 修改 roadmap
- 是我没时间吗？→ 砍范围，保留最小可用
- 是我在逃避吗？→ 跟点灯人聊一下

---

*Roadmap 不是契约，是当前最好的估计。修改它不是失败，是诚实。*
